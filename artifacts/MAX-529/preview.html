/orchestrator

Keep AI teams moving, route work correctly, and require proof before anything counts as done.

AI teams usually fail because they go idle, pick the wrong next ticket, and ship without receipts. /orchestrator is the workflow engine that fixes those three problems with ticket contracts, domain-aware execution, independent verification, and proof packs.

Sweep mode

Process actionable tickets in priority order with contract validation and independent verification.

Team mode

Run a parallel roster for sprint work so specialists never sit idle.

Proof pack

Artifact link, evidence log, run output, canonical link, and verification report.

Guardrails

Battle-tested hard rules written after real incidents, not generic process advice.

The 5 hard rules

  1. Never re-dispatch an In Progress ticket (incident: 2026-03-10)
  2. Never merge PRs (battle-tested across multiple incidents)
  3. Never modify your own cron job (incident: 2026-02-25)
  4. Never post status chatter without a delta (incident: 2026-03-11)
  5. Parent sweep dispatches only, never implements (incident: 2026-03-10)
