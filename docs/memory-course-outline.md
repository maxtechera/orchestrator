## Module 1. Why agents forget
### Outcome
Understand why context windows, compaction, and session loss break otherwise-good agent workflows.
### Lessons
1. Context windows are not memory
2. Why compaction loses the exact details you need later
3. Session loss, retries, and handoff drift in real agent work
4. What “durable context” actually needs to preserve
### Exercise
Map a recent workflow failure to one of the four memory failure modes.

## Module 2. The 3-tier architecture
### Outcome
Design the HOT router, WARM topic memory, and COLD vault for a working memory stack.
### Lessons
1. HOT memory, what stays in-session and what gets routed out
2. WARM memory, topic files and working summaries
3. COLD memory, durable archives and source-of-truth notes
4. Routing rules, freshness, and retrieval paths
### Exercise
Draft a personal 3-tier map for one live project.

## Module 3. WAL protocol
### Outcome
Use write-ahead logging so critical facts survive compaction and crashes.
### Lessons
1. Why write-before-compaction matters
2. WAL entries, structure, and minimum metadata
3. Crash recovery and replay flows
4. Safe checkpoints before summarization
### Exercise
Write a WAL template for decisions, blockers, and next actions.

## Module 4. Install /memory + configure Obsidian
### Outcome
Get `/memory` running with Obsidian as the durable source of truth.
### Lessons
1. Install the free tool from GitHub
2. Folder structure and topic conventions
3. Obsidian vault setup for agent-readable notes
4. First sync and first successful recall
### Exercise
Connect one repo workflow to a vault and confirm notes appear in the right topic file.

## Module 5. Cross-platform sync
### Outcome
Move memory cleanly across Claude Code, OpenClaw, and Gemini CLI.
### Lessons
1. Shared memory conventions across tools
2. What travels well between assistants and what should not
3. Prompt contracts for writes, reads, and recalls
4. Handoff patterns for multi-tool continuity

Run one task across all three tools while preserving the same project memory.

## Module 6. TTL decay + the weekly /memory dream ritual
### Outcome
Keep memory fresh without letting the vault bloat or rot.
### Lessons
1. TTL decay for stale context
2. What to summarize, archive, or delete
3. Weekly dream ritual for consolidation
4. Measuring memory quality over time
### Exercise
Schedule and run the first weekly memory-maintenance ritual.
